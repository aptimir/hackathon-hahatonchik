# PDF -> Markdown Pipeline для хакатона ГПН

Локальный пайплайн для преобразования PDF-документов в Markdown с сохранением структуры: текста, таблиц и изображений.  
Проект также включает утилиты для сборки финального submission из разных прогонов.

---

## 1. Состав проекта

- `baseline_plus_docling_v6_1_no_ocr_clean.py` — основной конвертер PDF -> Markdown (основной пайплайн)
- `build_all_good_markers.py` — сборка и агрегация промежуточных результатов
- `build_final_bestshot.py` — финальная сборка лучшего submission

---

## 2. Требования

- Python 3.10+
- open-source зависимости (например: `docling`, `pillow`)

Установка:
```bash
pip install docling pillow
```

---

## 3. Запуск

### Основной пайплайн

```bash
python baseline_plus_docling_v6_1_no_ocr_clean.py \
  --input-dir ./data \
  --output-dir ./output \
  --device auto
```

Основные параметры:
- `--input-dir` — входные PDF  
- `--output-dir` — директория результата  
- `--device` — `auto / cpu / cuda`  
- `--skip-existing` — пропуск уже обработанных файлов  

---

## 4. Сборка промежуточных результатов

Используется для объединения и подготовки результатов из разных запусков:

```bash
python build_all_good_markers.py
```

---

## 5. Финальная сборка submission

```bash
python build_final_bestshot.py
```

---

## 6. Структура данных

### Вход
```
data/
├── document_001.pdf
```

### Выход
```
output/
├── document_001.md
└── images/
    ├── doc_1_image_1.png
```

---

## 7. Как работает пайплайн

```
PDF -> извлечение -> обработка -> постобработка -> Markdown
```

Кратко:
- извлекается текст, таблицы и изображения  
- выполняется очистка и нормализация  
- результат сохраняется в Markdown  

Дополнительно:
- возможна агрегация нескольких прогонов  
- финальный результат собирается отдельным скриптом  

---

## 8. Важные детали

- имена изображений строго соответствуют ТЗ:
  `doc_<id>_image_<n>.png`
- порядок элементов сохраняется как в PDF  
- удаляются колонтитулы и шум  
- битые ссылки на изображения очищаются  

---

## 9. Воспроизводимость

- полностью локальный запуск  
- без использования облачных API  
- одинаковый результат при одинаковых входных данных  

---

## 10. Итог

Проект представляет собой простой и воспроизводимый пайплайн PDF -> Markdown  
с возможностью сборки финального submission из нескольких прогонов.
