# PDF -> Markdown Pipeline для хакатона ГПН

Локальный пайплайн для преобразования PDF-документов в Markdown с сохранением структуры: текста, таблиц и изображений.  
Также содержит утилиты для выбора лучших результатов и сборки финального submission.

---

## 1. Состав проекта

- `baseline_plus_docling_v5_5.py` — основной конвертер PDF → Markdown  
- `router_v2.py` - выбор лучшей версии документа из нескольких результатов  
- `submission_swapper.py` — сборка submission-вариантов  

---

## 2. Требования

- Python 3.10+
- open-source зависимости (например: `docling`, `pillow`)

Установка:
```bash
pip install docling pillow
```

---

## 3. Запуск

### Основной пайплайн

```bash
python baseline_plus_docling_v5_5.py \
  --input-dir ./data \
  --output-dir ./output \
  --device auto
```

Основные параметры:
- `--input-dir` — входные PDF  
- `--output-dir` — директория результата  
- `--device` — `auto / cpu / cuda`  
- `--skip-existing` — пропуск уже обработанных файлов  

---

## 4. Роутер (опционально)

Используется, если есть несколько прогонов и нужно выбрать лучший результат:

```bash
python router_v2.py \
  --output-all ./output_all \
  --output-v55 ./output_v5_5 \
  --output-noocr ./output_noocr \
  --dest ./routed_output
```

---

## 5. Сборка submission

```bash
python submission_swapper.py \
  --config ./swap_config.json \
  --output-root ./submission_variants
```

## 6. Структура данных

### Вход
```

├── document_001.pdf
```

### Выход
```
output/
├── document_001.md
└── images/
    ├── doc_1_image_1.png
```

---

## 7. Как работает пайплайн

```
PDF -> извлечение -> OCR / no-OCR -> постобработка -> выбор лучшего -> Markdown
```

Кратко:
- генерируются версии с OCR и без  
- выполняется очистка и нормализация  
- выбирается лучший результат по эвристикам  

---

## 8. Важные детали

- имена изображений строго соответствуют ТЗ:
  `doc_<id>_image_<n>.png`
- порядок элементов сохраняется как в PDF  
- удаляются колонтитулы и шум  
- битые ссылки на изображения очищаются  

---

## 9. Воспроизводимость

- полностью локальный запуск  
- без использования облачных API  
- одинаковый результат при одинаковых входных данных  

---

## 10. Итог

Проект представляет собой простой и воспроизводимый пайплайн PDF -> Markdown  
с базовой обработкой сложных элементов и инструментами для сборки финального submission.
